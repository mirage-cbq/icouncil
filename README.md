# AI Council — 多角色AI评议系统

> 仿三省六部制的多AI协同决策评估框架

## 核心理念

AI Council 将一个决策议题交由多个独立AI角色从不同视角审查，经过起草、批驳、专科审查、交叉辩论、整合汇总五个阶段，最终输出结构化评议报告，供人类参考决策。

系统不替人做决定，只负责把问题想透。

---

## 系统架构

```
人类提交议案
     ↓
【准备层】议案解构官
     ↓
【评议层】各角色独立审查（隔离）
  ├── 起草官
  ├── 批驳官
  └── 专科顾问（动态加载）
     ↓
【辩论层】交叉质询（最多3轮）
     ↓
【整合层】整合官汇总
     ↓
【输出层】评议报告 → 人类决策
```

## 角色说明

| 角色 | 职责 | Temperature |
|------|------|-------------|
| 议案解构官 | 标准化议案，识别评估维度 | 0.3 |
| 起草官 | 补全方案细节，建设性视角 | 0.8 |
| 批驳官 | 专职找漏洞和风险，找不到问题视为失职 | 0.3 |
| 专科顾问 | 特定领域审查（法律/财务/技术/伦理等） | 0.5 |
| 整合官 | 汇总争议，输出结构化报告 | 0.2 |

---

## 评议流程规则

- **隔离原则**：评议层各角色互相不可见，独立输出
- **强制异见**：批驳官必须提出异议，无异议视为失职
- **辩论上限**：每轮辩论最多3轮，防止死循环
- **补充调查**：整合官发现信息缺口可触发补充，最多2次
- **共识预警**：各角色高度一致时自动追加反向挑战轮

---

## 输出报告结构

- 结论摘要 + 置信度等级
- 支持理由（含来源角色）
- 反对/风险意见（含来源角色）
- 未解决的核心争议
- 信息缺口说明
- 完整评议过程（可展开）

## 快速开始

```bash
# 安装依赖
pip install -r requirements.txt

# 提交议案
python main.py --topic "你的议题"

# 指定角色模型（可选）
python main.py --topic "你的议题" --config config/custom.yaml
```

---

## 设计原则

1. 每一层只做一件事
2. 角色之间信息流向单向受控
3. 系统只输出判断依据，不替人决策
4. 任何环节的结论都可追溯到具体角色和论据
