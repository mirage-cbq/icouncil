## 工具定义规范

每个工具在接入时需提供：

```yaml
tool_id: web_search
name: 网络搜索
description: 搜索互联网获取最新信息
input_schema:
  query: string        # 搜索关键词
  max_results: int     # 最大返回数量，默认5
output_schema:
  results:
    - title: string
      url: string
      snippet: string
allowed_roles:
  - decomposer
  - drafter
  - critic
  - all_specialists
  - integrator
```

## 当前已定义工具

### 通用工具

**web_search** — 网络搜索
- 用途：获取最新信息、验证数据
- 全角色可用

**knowledge_base_search** — 知识库检索
- 用途：检索项目内部知识库
- 全角色可用

**case_history_search** — 历史案例查询
- 用途：检索类似历史议案及其结果
- 可用角色：解构官、起草官、批驳官

---

### 批驳官专用工具

**counterexample_search** — 反例检索
- 用途：寻找方案失败的历史案例

**logic_checker** — 逻辑一致性检验
- 用途：检验论证链条是否存在逻辑谬误

**risk_database** — 风险数据库查询
- 用途：查询特定领域的已知风险模式

---

### 专科顾问工具

**legal_database** — 法规数据库（法律顾问）
- 用途：查询法律法规、判例

**financial_model** — 财务模型（财务顾问）
- 用途：计算财务指标、查询市场数据

**tech_docs_search** — 技术文档检索（技术顾问）
- 用途：查询技术规范、开源生态信息

**ethics_framework** — 伦理框架参考（伦理顾问）
- 用途：对照伦理准则评估方案

---

### 整合官专用工具

**data_validator** — 数据验证
- 用途：交叉验证各角色引用的数据是否一致

**literature_search** — 文献检索
- 用途：查找学术或权威来源补充信息缺口

---

## 新增工具流程

1. 在本文件中按规范定义工具
2. 在对应角色的 `.md` 文件中更新"可调用工具"列表
3. 在 `council-config.yaml` 中注册工具配置
