# 主流程说明

## 流程状态机

```
IDLE
  ↓ 收到议案
DECOMPOSING          # 准备层：议案解构
  ↓
REVIEWING            # 评议层：并行独立审查
  ↓
DEBATING             # 辩论层：交叉质询
  ↓
INTEGRATING          # 整合层：汇总评估
  ↓ (需要补充?)
SUPPLEMENTING        # 补充调查（可选，最多2次）
  ↓
REPORTING            # 生成最终报告
  ↓
DONE
```

## 各阶段输入/输出

| 阶段 | 输入 | 输出 |
|------|------|------|
| DECOMPOSING | 人类原始议案 | 标准化任务包 |
| REVIEWING | 任务包 | 各角色独立意见 |
| DEBATING | 各角色意见 | 辩论记录 + 收敛状态 |
| INTEGRATING | 辩论记录 | 汇总评估 + 是否回流 |
| SUPPLEMENTING | 信息缺口列表 | 补充信息 |
| REPORTING | 全部过程数据 | 最终评议报告 |

## 回流条件

整合层在以下情况触发回流至 REVIEWING：
1. 发现关键信息缺口且可通过工具获取
2. 各方对同一事实存在不同认定（需要核实）

回流计数器上限：2次。超过后强制进入 REPORTING，在报告中标注"信息不完整"。

## 全局终止保障

- 单次评议最大轮次：可在配置中设置
- 任意阶段超时 → 强制进入 REPORTING，标注当前进度
- 输出置信度低于 0.3 的角色意见自动降权并标注
